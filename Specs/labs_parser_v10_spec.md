# Lab Parser v10 Spec — MCP Server

Related: [[labs_parser_roadmap]] · [[labs_parser_v9_spec]]

By v9 my parser is a deployed web app. v10 is the version that closes the loop — I expose the parser as an MCP server so Claude (in Cowork, Claude Desktop, Claude Code, anywhere) can call it directly. "Claude, what's my latest LDL?" → Claude calls my MCP → Claude gets the answer. The thing I've been using for a year, I'm now building.

This is the highest-leverage portfolio version because almost no one has shipped a real MCP server yet, and it makes the project useful in my actual daily workflow.

## What I Want to Learn

- Building an MCP server in Python with the official SDK
- Tool registration, resource registration, prompt templates
- Async server design (MCP servers are async by default)
- Local vs remote MCP servers — `stdio` transport vs HTTP transport

## What MCP Actually Is

MCP = Model Context Protocol. It's a JSON-RPC protocol that lets a client (Claude Desktop, Cowork, etc.) discover and call tools/resources/prompts hosted by a server. Three primitives:

- **Tools** — functions the LLM can call (like Anthropic API tool use, but standardized across hosts)
- **Resources** — data the LLM can read (like files or DB rows, addressable by URI)
- **Prompts** — reusable prompt templates the user can invoke as slash commands

Think of it as: my v7 REST API, but with a schema discoverable by any LLM client, and Claude knows how to use it without me writing a connector for each host.

## Two Ways to Run This

| Mode | Transport | Use case |
|------|-----------|----------|
| Local | stdio (subprocess) | I add it to Claude Desktop's config, runs on my Mac, hits localhost API |
| Remote | HTTP (Streamable HTTP) | Hosted on Render alongside v9 backend, anyone with auth can use it |

v10 ships both. The MCP server class is the same code; only the transport setup differs.

## Tools to Expose

Same conceptual tools as v3, but now exposed via MCP so Claude can call them from outside my parser script:

| Tool | What it does |
|------|--------------|
| `get_latest_draw` | Returns the most recent lab draw + flagged values |
| `get_marker_history` | Full history of one marker, optionally with date range |
| `compare_draws` | Diff two draws by ID, return what changed |
| `lookup_optimal_range` | Same as v3 — Haiku-powered functional optimal lookup |
| `analyze_draw` | Run the full v3 tool use analysis on a draw, return text summary |
| `import_lab_file` | Accept a file path or base64 PDF, ingest it (write tool, requires confirmation) |

## Resources to Expose

Resources are read-only data accessed by URI. Examples:

```
labs://draws/latest                  → JSON of latest draw
labs://draws/{id}                    → JSON of a specific draw
labs://markers/{name}/history        → JSON history of a marker
labs://summary/quarterly             → autogenerated quarterly summary
```

When Claude is asked "summarize my health this quarter," it reads these resources directly without needing to call a tool.

## Prompts to Expose

Reusable templates that show up as slash commands in Claude Desktop:

```
/labs-quarterly-review     → fetches last 3 months, runs full analysis
/labs-explain {marker}     → educates on what a marker is + my history with it
/labs-compare {id1} {id2}  → diff two draws with commentary
```

## Server Skeleton (Python MCP SDK)

```python
from mcp.server.fastmcp import FastMCP

mcp = FastMCP("labs-parser")

@mcp.tool()
async def get_latest_draw() -> dict:
    """Return the most recent lab draw with flagged values."""
    draw = await db.get_latest_draw()
    return draw.model_dump()

@mcp.tool()
async def get_marker_history(marker: str, days: int = 365) -> dict:
    """Return historical values for a marker over N days."""
    return await db.get_marker_history(marker, days)

@mcp.resource("labs://draws/latest")
async def latest_draw_resource() -> str:
    draw = await db.get_latest_draw()
    return draw.model_dump_json(indent=2)

@mcp.prompt()
def quarterly_review() -> str:
    return "Pull the last 3 months of lab draws using get_marker_history for key markers (LDL, HDL, ApoB, Lp(a), hsCRP, fasting glucose). Identify trends, flag anything moving in a concerning direction, and propose interventions."

if __name__ == "__main__":
    mcp.run()  # stdio transport by default
```

That's the local-mode server. ~30 lines for a working MCP. The HTTP version swaps `mcp.run()` for `mcp.run(transport="streamable-http", port=8001)`.

## Sharing Code with v7/v9

The MCP server reuses everything from v7's `core/` and `db.py`. New code is tiny — it's mostly a thin MCP wrapper around existing functions. That's the right architecture: same business logic, multiple interfaces (CLI, REST, MCP).

```
labs_parser/
├── core/             # shared
├── db.py             # shared
├── api/              # FastAPI from v7
├── cli.py            # argparse from v5
└── mcp_server.py     # NEW — MCP wrapper
```

## Registering with Claude Desktop

Add to `~/Library/Application Support/Claude/claude_desktop_config.json`:

```json
{
  "mcpServers": {
    "labs-parser": {
      "command": "uv",
      "args": ["run", "--directory", "/path/to/labs-parser", "python", "-m", "labs_parser.mcp_server"]
    }
  }
}
```

Restart Claude Desktop. The tools show up. Now I can ask "what's my latest LDL" in any Claude chat and it works.

## Registering Remote (HTTP) Mode

Hosted alongside v9 backend on Render at `labs-parser.onrender.com/mcp`. Auth via OAuth or a bearer token (the MCP spec supports both).

For v10 I'll use a bearer token — simpler, fine for single-user. OAuth comes if I ever multi-tenant this (out of scope).

In Cowork or Claude Desktop, add the remote MCP via the connector UI with my token.

## Write Operations: Confirmation Pattern

`import_lab_file` is a write tool. MCP clients show it differently to the user — Claude Desktop pops a confirmation. The tool description should make the destructive nature clear:

```python
@mcp.tool()
async def import_lab_file(file_path: str) -> dict:
    """Import a CSV or PDF lab file into the database. WRITE OPERATION — adds new records."""
```

## Edge Cases

| Case | Behavior |
|------|----------|
| Claude asks for a marker that doesn't exist | Tool returns `{"error": "marker not found", "available": [...]}` |
| Database connection lost | MCP server returns structured error, Claude can retry |
| User imports same file twice via MCP | Same dedup as v5 — return "already imported" |
| MCP server crashes | stdio transport: Claude Desktop restarts subprocess. HTTP: Render restarts container. |
| Stale token on remote MCP | Server returns 401, Claude prompts user to re-auth |
| Tool returns >1MB result | Truncate + add `truncated: true` field, link to full data via resource URI |
